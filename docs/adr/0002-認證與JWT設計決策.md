# ADR 0002:認證與 JWT 設計決策(M1)

日期:2026-07-11|狀態:已採納

## 背景

M1 實作認證模組(register / login / refresh)、Spring Security + JWT、RBAC、refresh token 可撤銷。設計文件第 9、10 節與 CLAUDE.md 定了大方向,但多處實作細節需就地決策。

## 決策

### 1. Token 型別與撤銷機制
- access(15m)/ refresh(7d)皆為 HS256 JWT,各帶 `type` claim(access / refresh)防混用。
- refresh 帶 `jti`(UUID);登入時以 `auth:refresh:{userId} = jti`(TTL = refresh 有效期)存入 Redis。
- 換發只驗證 jti 是否與 Redis 現存值相符;logout 直接刪 key 完成撤銷。
- **取捨**:每個 userId 僅保留一個有效 refresh,同一使用者多裝置登入會互相覆蓋。MVP 可接受;多裝置支援(改存 `auth:refresh:{userId}:{jti}` set)留待後續。

### 2. 401 vs 403 與統一格式
- 未認證/憑證無效 → 401 `code=1401`(`RestAuthenticationEntryPoint`)。
- 已認證但權限不足 → 403 `code=1403`(`RestAccessDeniedHandler`)。
- 兩者都在 Security filter 層,不經 `@RestControllerAdvice`,故各自寫 JSON;controller 路徑上的例外才由 `GlobalExceptionHandler` 處理。
- token 缺失/過期/篡改一律由 filter「靜默不認證」,交授權層回 401(三種場景測試皆通過),不外洩失敗原因。

### 3. 帳號枚舉防護
- 登入時「帳號不存在 / 密碼錯誤 / 帳號停用」一律回同一個 `1002 帳號或密碼錯誤`,避免枚舉。

### 4. Snowflake 細節(補充設計文件)
- 設計文件僅指定 `WORKER_ID` 環境變數;datacenterId 同樣以 `DATACENTER_ID` 環境變數提供(預設 0),維持 5+5 bit 結構。
- 自訂 epoch `2026-01-01T00:00:00Z`;`nextId()` 為 `synchronized`;時鐘回撥 ≤5ms 自旋、>5ms 拋例外並累加 `id_generator_clock_backwards_total`。
- `currentTimeMillis()` 抽為 protected 方法以利測試模擬回撥。

### 5. ID 序列化為字串
- 64-bit Snowflake ID 在回應 DTO 中一律以 **String** 回傳(userId 等),避免前端 JS `number` 超過 2^53 損失精度。

### 6. 密碼強度
- username 3–50 字(英數底線);密碼 8–72 字且至少含一字母一數字(72 為 BCrypt 位元組上限)。
- **取捨**:ASVS 5.0 建議至少 12 字;此處放寬為 8 供 demo 便利,正式環境可調嚴(改 `RegisterRequest` 的 `@Size`)。

### 7. 建置:Lombok annotation processor
- JDK 23 起 javac 不再自動執行「僅存在於 classpath」的註解處理器,必須將 Lombok 明列進 `maven-compiler-plugin` 的 `annotationProcessorPaths`,否則 `@Data` 不生效。

### 8. 整合測試容器策略(修正 M0 做法)
- 改採 **singleton container** 模式:容器在 static 區塊啟動、不交給 `@Testcontainers` 管理生命週期,跨測試類別共用,JVM 結束由 Ryuk 清理。
- **原因**:M0 的 `@Testcontainers` + static `@Container` 在多測試類別下,先跑完的類別會於 afterAll 停掉共用容器,導致後續類別沿用快取 context 卻連到已停止容器(HealthEndpointIT 出現 503)。
- **取捨**:不再有「無 Docker 自動跳過」;整合測試現在硬性需要 Docker(符合設計文件「整合測試一律 Testcontainers、禁 H2」)。

## 影響
- 新增依賴使用:jjwt(已在 M0 pom)、Bucket4j 尚未啟用(M3)。
- `application.yml` 新增 `seckill.id.*`、`seckill.jwt.*`、`seckill.cors.allowed-origins`,全部走環境變數,無祕密入庫。
