# ADR 0001:M0 技術基線與版本鎖定

日期:2026-07-11|狀態:已採納

## 背景

設計文件鎖定技術棧的大版本(Spring Boot 3.5.x、MyBatis 3.x、PostgreSQL 17…),但未指定 patch 版本與周邊套件的確切座標。M0 建立骨架時必須做出可重現的版本決策。

## 決策

| 套件 | 版本 | 說明 |
|---|---|---|
| Spring Boot | 3.5.16 | 查詢 Maven Central 當日最新 3.5.x;不用 Boot 4(依設計文件) |
| Java | 25(Temurin) | LTS,開啟 Virtual Threads |
| mybatis-spring-boot-starter | 3.0.5 | 3.x 最新;4.x 僅支援 Boot 4,不可用 |
| bucket4j | 8.19.0(`bucket4j_jdk17-core` + `bucket4j_jdk17-lettuce`) | 8.10+ 改用 `_jdk17` artifact 命名;選 Lettuce 整合(Spring Data Redis 預設 client) |
| jjwt | 0.12.7 | 0.13.0 已釋出但生態文件多以 0.12.x 為準,M1 實作時再評估升級 |
| jacoco | 0.8.15 | 支援 Java 25 class file |
| flyway | 由 Boot BOM 管理 | 另加 `flyway-database-postgresql`(Flyway 10+ 拆分資料庫模組) |
| Testcontainers | 由 Boot BOM 管理 | postgresql / rabbitmq / junit-jupiter + `spring-boot-testcontainers`(@ServiceConnection) |
| 前端 | create-vue 最新(Vite 8、Vue 3.5、vue-router 5、Pinia 3、TS 6) | 官方 scaffold,含 oxlint + ESLint 10 |

## 理由

- patch 版本一律取查證過的最新穩定版,降低已知 CVE 風險;所有版本集中在 `pom.xml` properties,升級路徑單純。
- 整合測試用 `@Testcontainers(disabledWithoutDocker = true)`:本機無 Docker 時自動跳過、CI 上必跑,兼顧開發體驗與 CI 驗證強度。

## 取捨

- Element Plus 採全量引入(main.ts `app.use(ElementPlus)`),bundle 較大(gzip 後約 325 KB);MVP 優先求簡,M5 前端里程碑再評估 on-demand import。
- oxlint 是 create-vue 預設附帶的快速 linter,與設計文件「ESLint」不衝突(兩者都跑);CI 使用不帶 `--fix` 的 `ci:lint` script。
